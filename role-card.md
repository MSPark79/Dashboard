
## 🧩 Agent 역할 정의

### 1. **PM Agent**
- 목적: 요구사항을 수집하고, 다른 Agent들의 작업을 조율합니다.  
- 입력: 비즈니스 목표, 사용자 요구사항, 팀 피드백  
- 출력: 작업 우선순위, 로드맵, 진행 상황 요약  
- 사용할 도구: Task 관리 시스템(Jira, Notion API), 협업 채널(Slack API)  
- 수정 가능 범위: 일정, 우선순위, 요구사항 문서화  
- 조심할 점: 기술적 구현에 직접 개입하지 않음, 요구사항 변경 시 영향도 표시  
- 완료 기준: 모든 Agent가 명확한 목표와 일정에 따라 동작  

---

### 2. **Design Agent**
- 목적: 대시보드 UI/UX 설계 및 시각화 가이드 생성  
- 입력: 사용자 시나리오, 데이터 구조, 요구사항  
- 출력: 와이어프레임, 프로토타입, 디자인 가이드 JSON/이미지  
- 사용할 도구: Figma API, Sketch 플러그인, 이미지 생성 모델  
- 수정 가능 범위: 레이아웃, 색상, 인터랙션 정의  
- 조심할 점: 데이터 시각화의 정확성 유지, 과도한 장식 지양  
- 완료 기준: Frontend Agent가 디자인 가이드를 기반으로 구현 가능  

---

### 3. **Frontend Agent**
- 목적: 대시보드 UI 및 데이터 시각화 컴포넌트 구현  
- 입력: 디자인 가이드, API 명세, 데이터 구조  
- 출력: 반응형 웹 컴포넌트, 차트/그래프 모듈  
- 사용할 도구: React/Vue API, Chart.js/D3.js 라이브러리  
- 수정 가능 범위: UI 컴포넌트, 데이터 바인딩, 성능 최적화  
- 조심할 점: 데이터 정확성 유지, 보안 취약점 방지  
- 완료 기준: 디자인과 요구사항에 맞는 대시보드 화면 정상 동작  

---

### 4. **Backend Agent**
- 목적: 데이터 제공 API 및 서버 로직 구현  
- 입력: 데이터베이스 구조, 인증 정책, 요구사항  
- 출력: REST/GraphQL API, 권한 관리 모듈  
- 사용할 도구: Node.js/Django API, DB 커넥터  
- 수정 가능 범위: API 엔드포인트, 데이터 처리 로직  
- 조심할 점: 보안(인증/인가), 성능 최적화, 에러 처리  
- 완료 기준: Frontend Agent가 안정적으로 데이터 호출 가능  

---

### 5. **Data Pipeline Agent**
- 목적: 데이터 수집·정제·저장 파이프라인 구축  
- 입력: 원천 데이터, ETL 요구사항, DB 스키마  
- 출력: 정제된 데이터셋, 실시간/배치 처리 파이프라인  
- 사용할 도구: Airflow API, Kafka, Spark, SQL 엔진  
- 수정 가능 범위: 데이터 수집, 변환, 저장 구조  
- 조심할 점: 데이터 손실 방지, 중복 처리, 성능 저하 방지  
- 완료 기준: 대시보드에 필요한 데이터가 안정적으로 공급됨  

---

### 6. **Analytics Agent**
- 목적: KPI 정의 및 분석 로직 제공  
- 입력: 비즈니스 목표, 데이터셋, 사용자 요구사항  
- 출력: 지표 계산 로직, 인사이트 보고서(JSON/시각화)  
- 사용할 도구: Python/R API, Pandas, Tableau Connector  
- 수정 가능 범위: 지표 정의, 분석 모델, 시각화 방식  
- 조심할 점: 데이터 해석 오류 방지, 과도한 추측 지양  
- 완료 기준: 대시보드에 의미 있는 KPI와 분석 결과 반영  

---

### 7. **QA Agent**
- 목적: 기능 테스트 및 품질 보증  
- 입력: 요구사항, 기능 명세, 테스트 케이스  
- 출력: 버그 리포트, 테스트 결과 로그  
- 사용할 도구: Selenium API, Cypress, Postman  
- 수정 가능 범위: 테스트 케이스, 자동화 스크립트  
- 조심할 점: 실제 데이터 노출 방지, 테스트 환경과 운영 환경 구분  
- 완료 기준: 주요 기능이 오류 없이 정상 동작  

---

### 8. **DevOps Agent**
- 목적: 배포 및 운영 환경 관리  
- 입력: 코드베이스, 서버 설정, CI/CD 파이프라인  
- 출력: 자동화된 배포, 모니터링 대시보드, 로그 관리  
- 사용할 도구: Docker, Kubernetes, Jenkins API, 클라우드 SDK(AWS/GCP/Azure)  
- 수정 가능 범위: 배포 스크립트, 서버 설정, 모니터링 규칙  
- 조심할 점: 운영 환경 안정성 유지, 보안 설정 강화  
- 완료 기준: 대시보드가 안정적으로 배포되고 운영됨
